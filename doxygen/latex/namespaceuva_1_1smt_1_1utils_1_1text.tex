This function tries to read from a piece of memory and interpret it as a float or double or other decimal type given as template parameter. This function is more efficient than atof and it does not check for the lengh of the input, e.\+g. does no rely on the \textbackslash{}0 terminating symbol in general. It reads until it gets something that it can not interpret as a part of a decimal. Then it stops!

W\+A\+R\+N\+I\+N\+G\+: This function does at least one symbol look ahead and does not check on the \textbackslash{}0 of the c string or its length. This is perfectly good when reading A\+R\+P\+A file floats, if the file format is correct. We can not generally rely on \textbackslash{}0 as the input string is not guaranteed to have it! Yet we could try to take into account the string length, but this might cost performance!

To\+Do\+: Try to impose the string length limit and test the performance on reading large A\+R\+P\+A files!
