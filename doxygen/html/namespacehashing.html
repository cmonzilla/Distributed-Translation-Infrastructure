TWordHashSize hashing::computeDjb2Hash ( const string &  str)
inline

This is one of the best known hashing function algorithms (djb2) for the C strings as reported and described in http://www.cse.yorku.ca/~oz/hash.html Note: It turned to be not as good as the PrimesHash as resulted in collisions on the test data.

Note: The time complexity of this algorithm is also linear in the length of the input word.

This function will combine two word references to get one hash map N-gram level reference.

This is a Szudzik's function used for pairing. http://szudzik.com/ElegantPairing.pdf This function is more efficient that than of Cantor as it uses indexes densely the result of szudzik(N,M) is <= 2*max(M,N), so the risk of owerflows is much smaller!
