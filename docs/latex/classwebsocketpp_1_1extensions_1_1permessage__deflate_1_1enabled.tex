Limit client L\+Z77 sliding window size. 

The bits setting is the base 2 logarithm of the window size that the client must use to compress outgoing messages. The permitted range is 8 to 15 inclusive. 8 represents a 256 byte window and 15 a 32\+Ki\+B window. The default setting is 15.

Mode Options\+:
\begin{DoxyItemize}
\item accept\+: Accept whatever the remote endpoint offers.
\item decline\+: Decline any offers to deviate from the defaults
\item largest\+: Accept largest window size acceptable to both endpoints
\item smallest\+: Accept smallest window size acceptiable to both endpoints
\end{DoxyItemize}

This setting is dependent on client support. A client may limit its own outgoing window size unilaterally. A server may only limit the client\textquotesingle{}s window size if the remote client supports that feature.

Limit server L\+Z77 sliding window size. 

The bits setting is the base 2 logarithm of the maximum window size that the server must use to compress outgoing messages. The permitted range is 8 to 15 inclusive. 8 represents a 256 byte window and 15 a 32\+Ki\+B window. The default setting is 15.

Mode Options\+:
\begin{DoxyItemize}
\item accept\+: Accept whatever the remote endpoint offers.
\item decline\+: Decline any offers to deviate from the defaults
\item largest\+: Accept largest window size acceptable to both endpoints
\item smallest\+: Accept smallest window size acceptiable to both endpoints
\end{DoxyItemize}

This setting is dependent on server support. A client requesting this setting may be rejected by the server or have the exact value used adjusted by the server. A server may unilaterally set this value without client support.
