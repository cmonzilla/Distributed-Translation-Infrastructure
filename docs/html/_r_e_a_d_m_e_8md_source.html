7 ##Introduction
8 This is a fork project from the Back Off Language Model(s) for SMT project aimed at creating the entire phrase-based SMT translation infrastructure. This project follows a client/server atchitecture based on WebSockets for C++ and consists of the three main applications:
9 
10 + **bpbd-client** - is a thin client to send the translation job requests to the translation server and obtain results
11 + **bpbd-server** - the the translation server consisting of the following main components:
12  - *Decoder* - the decoder component responsible for translating text from one language into another
13  - *LM* - the language model implementation allowing for seven different trie implementations and responsible for estimating the target language phrase probabilities.
14  - *TM* - the translation model implementation required for providing source to target language phrase translation and the probailities thereof.
15  - *RM* - the reordering model implementation required for providing the possible translation order changes and the probabilities thereof
16 + **lm-query** - a stand-alone language model query tool that allows to perform labguage model queries and estimate the joint phrase probabilities.
17 
18 To keep a clear view of the used terminology further we will privide some details on the phrase based statistical machine translation as given on the picture below.
19 
20 ![Statistical Machine Translation Approach Image](https://github.com/ivan-zapreev/Basic-Phrase-Based-Decoding/blob/master/docs/images/smt/smt.jpg "Statistical Machine Translation Approach")
21 
22 The entire phrase-based statistical machine translation is based on learned statistical correlations between words and phrases of an example translation text, also called parallel corpus or corpora. Clearly, if the training corpora is large enough then it allows to cover most source/target language words and phrases and shall have enough information for approximating a translation of an arbitrary text. However, before this information can be extracted, the parallel corpora undergoes the process called _word alignment_ which is aimed at estimating which words/phrases in the source language correspond to which words/phrases in the target language. As a result, we obtain two statistical models:
23 
24 1. The Translation model - providing phrases in the source language with learned possible target language translations and the probabilities thereof.
25 2. The Reordering model - storing information about probable translation orders of the phrases within the source text, based on the observed source and target phrases and alignment thereof.
26 
27 The last model, possibly learned from a different corpus in a target language, is the Language model. Its purpose is to reflect the likelihood of this or that phrase in the target language to occur. In other words it is used to evaluate the obtained translation for being _sound_ in the target language.
28 
29 With these three models at hand one can perform decoding, which is a synonim to a translation process. SMT decoding is performed by exploring the state space of all possible translations and reorderings of the source language phrases within one sentence and then looking for the most probable translations, as indicated at the bottom part of the picture above.

74 ##Building the project
75 Building this project requires **gcc** version >= *4.9.1* and **cmake** version >= 2.8.12.2. The project can be build in two ways:
76 
77 + From the Netbeans environment by running Build in the IDE
78  - Perform `mkdir build` in the project folder.
79  - In Netbeans menu: *Tools/Options/"C/C++"* make sure that the cmake executable is properly set.
80  - Netbeans will always run cmake for the DEBUG version of the project
81  - To build project in RELEASE version use building from Linux console
82 + From the Linux command-line console perform the following steps
83  - `cd [Project-Folder]`
84  - `mkdir build`
85  - `cd build`
86  - `cmake -DCMAKE_BUILD_TYPE=Release ..` OR `cmake -DCMAKE_BUILD_TYPE=DEBUG ..`
87  - `make -j [NUMBER-OF-THREADS]` add `VERBOSE=1` to make the compile-time options visible
88 
89 The binaries will be generated and placed into *./build/* folder. In order to clean the project from the command line run `make clean`.

145 ###Language model
146 For machine translation it is important to estimate and compare the fluency of different possible translation outputs for the same source (i.e., foreign) sentence. This is commonly achieved by using a language model, which measures the probability of a string (which is commonly a sentence). Since entire sentences are unlikely to occur more than once, this is often approximated by using sliding windows of words (n-grams) occurring in some training data.
147 
148 ####Language Models background
149 An *n-gram* refers to a continuous sequence of n tokens. For instance, given the following sentence: `our neighbor , who moved in recently , came by .` If n = 3, then the possible n-grams of
150 this sentence include:
151 
152 ```
153 "our neighbor ,"
154 "neighbor , who"
155 ", who moved"
156 ...
157 ", came by"
158 "came by ."
159 ```
160 Note that punctuation marks such as comma and full stop are treated just like any _real_ word and that all words are lower cased.

182 In this section we mention a few implementation details, for more details see the source code documentation. The code contains the following important source files:
183 
184 * **main.cpp** - contains the entry point of the program
185 * **Executor.cpp** - contains some utility functions including the one reading the test document and performing the queries on a filled in Trie instance.
186 * **ARPATrieBuilder.hpp / ARPATrieBuilder.cpp** - contains the class responsible for reading the ARPA file format and building up the trie model using the ARPAGramBuilder.
187 * **TrieDriver.hpp** - is the driver for all trie implementations - allows to execute queries to the tries.
188 * **LayeredTrieDriver.hpp** - is a wrapper driver for all the layered trie implementations - allows to retrieve N-gram probabilities and back-off weights.
189 * **C2DHashMapTrie.hpp / C2DHashMapTrie.cpp** - contains the Context-to-Data mapping trie implementation based on unordered_map.
190 * **C2DMapArrayTrie.hpp / C2DMapArrayTrie.cpp** - contains the Context-to-Data mapping trie implementation based on unordered_map and ordered arrays.
191 * **C2WOrderedArrayTrie.hpp / C2WOrderedArrayTrie.cpp** - contains the Context-to-Word mapping trie implementation based on ordered arrays.
192 * **G2DHashMapTrie.hpp / G2DHashMapTrie.cpp** - contains the M-Gram-to-Data mapping trie implementation based on self-made hash maps.
193 * **W2CHybridMemoryTrie.hpp / W2CHybridMemoryTrie.cpp** - contains the Word-to-Context mapping trie implementation based on unordered_map and ordered arrays.
194 * **W2COrderedArrayTrie.hpp / W2COrderedArrayTrie.cpp** - contains the Word-to-Context mapping trie implementation based on ordered arrays.
195 * **Configuration.hpp** - contains configuration parameter for the word index and trie and memory management entities.
196 * **Exceptions.hpp** - stores the implementations of the used exception classes.
197 * **HashingUtils.hpp** - stores the hashing utility functions.
198 * ** ARPAGramBuilder.hpp / ARPAGramBuilder.cpp** - contains the class responsible for building n-grams from a line of text and storing it into Trie.
199 * **StatisticsMonitor.hpp / StatisticsMonitor.cpp** - contains a class responsible for gathering memory and CPU usage statistics
200 * **Logger.hpp/Logger.cpp** - contains a basic logging facility class
